# 淘宝导入 Detail 步骤返回确认弹窗

## 问题

在淘宝导入的 Detail 步骤中，用户花费大量时间编辑各个 Item 的品牌、分类、颜色等信息后，
不小心按了返回键（应用内返回箭头或系统返回键），所有编辑内容全部丢失，无法恢复。

## 方案

在 Detail 步骤拦截所有返回操作，弹出确认对话框，用户确认后才执行返回。

## 设计

### 触发条件

1. 点击 TopAppBar 左上角返回箭头
2. 按系统返回键（手势导航或虚拟按键）

### 交互流程

1. 用户触发返回 → 显示 AlertDialog
2. 标题：`确认返回`
3. 正文：`已编辑的导入数据尚未保存，返回后将丢失所有修改。`
4. 确认按钮：`返回`（执行 onBack）
5. 取消按钮：`继续编辑`（关闭对话框）

### 改动范围

仅 `ImportDetailScreen.kt`：
- 添加 `showBackConfirmDialog` 状态
- 返回箭头 onClick 改为显示弹窗
- 添加 `BackHandler` 拦截系统返回键
- 添加 `AlertDialog` composable
